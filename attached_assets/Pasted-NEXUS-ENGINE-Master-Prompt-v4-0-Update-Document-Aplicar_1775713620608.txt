STATE PERSISTENCE — CRITICAL
Zustand store with persist middleware writing to localStorage on EVERY state change. Opening any panel, section, or overlay during an active run must NEVER reset or clear the narrative history. Run state survives: panel switching, screen navigation, page refresh, browser close/reopen. Full run history persisted: all narration turns, events, character state, world state. On app load: check localStorage for active run → show “Continuar partida” if found. NEVER use React useState for active run data — only Zustand + localStorage. Manual save button always available during a run: saves everything explicitly to Supabase as backup.

Cada partida guardada es un objeto independiente con ID único. Al guardar se asigna un slot nuevo sin sobrescribir los existentes a menos que el jugador elija explícitamente “Sobreescribir” sobre una partida ya guardada.

AI MEMORY SYSTEM — CRÍTICO
La IA DEBE leer y respetar absolutamente TODOS los datos almacenados antes de generar cualquier narración. Si el jugador definió que su madre se llama Marthe Leonids, la IA NUNCA puede usar otro nombre. Si definió que nació en Vertain, la IA NUNCA puede ignorarlo. TODOS los datos definidos por el jugador son canónicos y tienen prioridad absoluta sobre cualquier generación propia.
Antes de cada llamada a la API, el motor inyecta automáticamente en el system prompt: ficha completa del personaje con todos sus campos tal como el jugador los definió, estado del mundo actual, fichas completas de todos los NPCs conocidos, fichas de todas las facciones conocidas, el historial completo de TODOS los turnos de la partida sin excepción, cola de consecuencias activa, inventario completo, descriptores actuales, clima emocional actual, últimas entradas de voz interior, año y fecha en juego actuales, y todas las instrucciones permanentes del jugador.
La IA mantiene una base de conocimiento interna que se actualiza automáticamente. Cada NPC que aparece queda registrado con toda su información y la IA nunca puede olvidarlo ni contradecirlo. Cada hecho establecido narrativamente (nacimientos, muertes, eventos, acuerdos, objetos obtenidos) se registra como hecho canónico y no puede ser contradicho. La IA nunca inventa datos que ya están definidos.
Panel “Memoria IA” accesible desde el juego como panel propio. Contiene: lista de todos los hechos canónicos establecidos, timeline completo en-juego con fecha y turno de cada evento, red de relaciones entre todos los personajes actualizada tras cada turno, campo de notas del narrador (texto libre, sin límite de caracteres, la IA DEBE leerlas en cada turno), campo de reglas personalizadas de partida (texto libre, sin límite, instruye a la IA sobre esa partida específica), campo de instrucciones permanentes globales (texto libre, sin límite, la IA las sigue en TODAS las partidas siempre).

Multi-step flow. Every step has: ← Back, Siguiente →, step indicator at top.
Step 0 — Mode Selection: Two large cards. MODO HUMANO: “Eres un personaje viviendo en el mundo. Tu perspectiva es la de un ser humano con limitaciones reales.” → lleva a creación de personaje. MODO DIOS: “Eres una entidad omnisciente. Observas, intervenes, y controlas el mundo desde fuera.” → lleva a World Genesis. Cada card tiene botón de información expandible.
Step 1 — Game Context: Muestra nombre del juego, frase atmosférica, descripción breve. Solo lectura.
Step 2 — Era & World: Toggle Mundo aleatorio / Mundo guardado. Selector Año de la era +/−. Fecha de nacimiento: día (1–31) + mes (nombres adaptados a la era). Toggles de reglas del mundo: Magia / Tecnología futurista / Desastres naturales / Guerra / Enfermedad. Reglas adicionales dinámicas según era. Slider MODO REALISMO 1–10. Notas adicionales del mundo: texto libre. “✦ Generar contexto del mundo”: llama a la API y muestra el resultado — MUST work.
Step 3 — Identity: Nombre (texto libre, placeholder: “Déjalo vacío para generación automática”). “✦ Sugerir nombre”: llama a API → devuelve 3 chips de nombres apropiados para la era. Género: dos botones fijos (Hombre / Mujer). Clase social: grid de botones filtrados por era. Riqueza inicial: slider (Miseria → Pobreza → Clase Media → Riqueza → Opulencia).
Step 4 — Appearance: Botón “✦ Generar completa” llama a API con era + clase + género y rellena todos los campos. Altura: dropdown (Muy bajo / Bajo / Promedio / Alto / Muy alto). Complexión: dropdown (Delgado / Esbelto / Atlético / Promedio / Robusto / Corpulento / Obeso). Tez: dropdown con opciones apropiadas a la era. Cabello: texto libre + “✦ Sugerir”. Ojos: dropdown. Rasgos distintivos: sistema de chips presets + campo para añadir chips propios sin límite máximo, cada chip con X para eliminar. Descripción libre: textarea opcional. Todo filtrado por era.
Step 5 — Personality: “✦ Generar personalidad” llama a API con todos los datos hasta aquí. Rasgos positivos (mín 1, máx 5): mínimo 15 chips preset. Rasgos negativos (mín 1, máx 5): mínimo 15 chips preset. Valores (mín 1, máx 3): mínimo 15 chips preset. Miedos (mín 1, máx 5): mínimo 20 chips preset. Motivaciones: texto libre + “✦ Sugerir”. Peculiaridades: texto libre + “✦ Sugerir”.
Step 6 — Skills & Background: Habilidades (mín 1, máx 7): mínimo 25 chips filtrados por era. Limitaciones (mín 1, máx 5): mínimo 15 chips. Historia de origen: textarea + “✦ Generar historia de origen” que llama a la API. Relaciones iniciales (hasta 5): cada una con campo de nombre + dropdown de tipo (mínimo 15 opciones: Madre / Padre / Hermano/a / Hijo/a / Abuelo/a / Tío/a / Primo/a / Amigo íntimo / Rival / Mentor / Amor / Enemigo / Aliado / Empleador / Vecino / Protector / Deudor…) + descripción breve + botón Agregar relación.
Step 7 — Beliefs: “✦ Sugerir creencias acordes a la época”. Religión / Fe: texto libre. Filosofía de vida: mínimo 15 chips con botón de info en cada uno. Visión política: chips preset + texto libre.
Step 8 — Final Review: Tarjeta resumen de todos los atributos. “✦ Generar narrativa de presentación” llama a API. Botón “COMIENZA LA EXISTENCIA” (verde, grande). Error si faltan campos obligatorios. Tap en cualquier sección del resumen vuelve a ese paso.

SALTAR TIEMPO — SISTEMA DE AVANCE TEMPORAL
Durante una partida activa, siempre disponible desde la pantalla de juego mediante un botón visible “⏩ Avanzar tiempo”, accesible también desde el panel de Personaje y desde el input de acciones.
Al activarlo, se abre un panel con las siguientes opciones:
AVANCE POR ETAPAS VITALES: Botones que saltan directamente a la siguiente etapa de vida del personaje. Las etapas disponibles dependen de la edad actual: Primera infancia (0–2 años), Infancia temprana (3–6 años), Infancia media (7–11 años), Adolescencia (12–17 años), Juventud (18–24 años), Adultez temprana (25–39 años), Madurez (40–59 años), Vejez (60+). El botón de cada etapa solo aparece si el personaje aún no ha llegado a ella. Se puede saltar una etapa o varias a la vez seleccionando el destino.
AVANCE POR AÑOS ESPECÍFICOS: Campo numérico donde el jugador escribe exactamente cuántos años quiere avanzar (mínimo 1, sin máximo salvo la edad de muerte natural estimada). Botón “Avanzar X años”.
AVANCE A AÑO CONCRETO: Campo para escribir el año en juego exacto al que se quiere llegar. El sistema calcula los años que faltan y muestra cuánta edad tendría el personaje.
Al confirmar cualquier avance: El sistema muestra un mensaje “Narrando X años de vida de [nombre]…” con animación de carga. La IA genera una narración extensa y coherente (mínimo 600 palabras) cubriendo todo el período saltado: eventos importantes, cambios de vida, relaciones que evolucionaron, habilidades adquiridas, pérdidas sufridas, momentos definitorios. La narración respeta TODOS los datos ya establecidos del personaje, el mundo y los NPCs. El motor actualiza automáticamente: edad del personaje, descriptores, inventario (añade o elimina items coherentes con el tiempo pasado), relaciones (los NPCs envejecen, algunos pueden morir si la edad lo justifica), habilidades (puede añadir nuevas aprendidas durante ese período), historial de personaje (se añaden entradas para el período), cola de consecuencias (las pendientes se resuelven o escalan). Si el período saltado incluye etapas donde se desbloquea la escritura libre (edad 13+), la narración lo menciona explícitamente y al terminar el juego retoma desde la edad de destino con todas las opciones desbloqueadas. La narración del salto se muestra en el área principal de juego como cualquier otra narración, con el mismo formato visual. El jugador puede hacer múltiples saltos en cualquier momento.
Durante el período de infancia (0–12), la pantalla de juego muestra un banner: “Edad actual: X años — [descripción de lo que puede hacer el jugador en esta etapa]” y un botón prominente “⏩ Avanzar a la siguiente etapa” para que sea visible y accesible sin necesidad de buscarlo.

La ficha de facción se llena progresivamente. TODO editable desde el Editor. Botón “Crear facción manualmente”. El motor añade facciones automáticamente cuando la narración las introduce.

NARRATIVE SYSTEMS
Age-based interaction: Edad 0: Sin interacción. Narración cinematográfica pura del nacimiento, el entorno, los padres, las primeras semanas. Mínimo 400 palabras. Completamente única cada vez. Edades 0–2: Narración pura. Sin input del jugador. Edades 3–6: Solo sugerencias. Sin texto libre. Edades 7–11: Texto libre con filtrado contextual intenso. Edades 12–17: Agencia más amplia, filtrada por contexto. Edad 18+: Agencia completa sin restricciones dentro de las reglas del mundo.
Durante las etapas 0–12 el sistema muestra el banner de etapa y el botón de avance temporal prominente en todo momento.
Calidad narrativa: Mínimo 200 palabras por respuesta. Primera narración: mínimo 500 palabras, larga, atmosférica, cinematográfica, siempre completamente única. Narración de salto temporal: mínimo 600 palabras cubriendo coherentemente todo el período. Nunca repetir la misma estructura narrativa, frase inicial o configuración de escena entre partidas. Variar estructura de frases, ritmo, vocabulario. Referenciar detalles específicos apropiados a la era. Rastrear y referenciar eventos recientes orgánicamente. Las acciones sugeridas son siempre generadas por IA, contextuales, nunca hardcoded, nunca repetidas.
Sistemas narrativos activos durante cada partida: Memoria Profunda (cada turno auto-etiquetado, recupera eventos pasados por resonancia temática, emocional, de ubicación, inyectados como ecos disponibles). Voz Interior (log de entradas del jugador, el narrador las saca en momentos emocionalmente resonantes, nunca citadas directamente). Clima Emocional (promedio progresivo, estados: Sereno / Ansioso / De duelo / Eufórico / Entumecido / Desesperado / Esperanzador / Traumatizado, cambia lentamente, afecta tono y acciones sugeridas). Otras Perspectivas (cuando activo, narrador cambia brevemente a perspectiva de NPC, tipográficamente distinto, solo en momentos dramáticamente relevantes). Ecos del Pasado (legado de partidas anteriores como textura narrativa únicamente, nunca anunciado). Secretos y Cosas No Dichas (capa oculta para eventos significativos, señales sutiles en la narración, visible completa en Modo Dios). Sueños (generados al dormir, siempre saltables, no siempre significativos, no dormir causa deterioro realista). Tiempo Subjetivo (invisible, solo sentido a través del ritmo de la prosa). El Mundo como Narrador (memoria de objetos, narración espacial, ironía ambiental, anclaje sensorial, ausencia como presencia). Consecuencias Diferidas (cola de consecuencias con timing realista, posible cascada, visible en Modo Dios y tab Legado).

CONFLICT RESOLUTION
Sin números, sin dados, sin porcentajes. Resultados narrados como consecuencias inevitables. Físico: cuerpo vs oponente, habilidades, estado físico, armas, entorno, estado psicológico. Social: clase relativa, reputación, historial de comunicación, normas culturales, estado emocional. Supervivencia: conocimiento + recursos + condición física + experiencia. Influencia del jugador: si es previsible el jugador puede prepararse en turnos anteriores con efecto real; si es sorpresa no hay ventana de preparación.

CULTURAL AND ERA COHERENCE
Moneda y transacciones solo apropiadas a la era con contexto de poder adquisitivo real. Vocabulario y conceptos solo disponibles en esa era. Normas sociales, clase, género definen restricciones realistas. Sistemas legales, castigos, propiedad específicos de la era. Conocimiento médico estrictamente apropiado a la era. Nombres de NPCs apropiados a la era y región. Ropa, comida, arquitectura, transporte: precisos a la era. Sin marcos psicológicos/sociológicos modernos en ambientaciones históricas. Nombres de meses y calendarios adaptados a la era.
